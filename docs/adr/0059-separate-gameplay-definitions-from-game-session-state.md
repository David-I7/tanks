# Separate gameplay content from game session state

The gameplay package becomes a deep module whose interface accepts versioned immutable Game Content plus the Game Session's mutable `World` and `TerrainModel`, then returns resolved state changes. `WorldDefinition` owns dimensions, Bedrock depth, tick rate, gravity, projectile simulation limits, and Spawn Regions; `TankDefinition` owns maximum resources, movement and fuel rules, climb capability, collision and track geometry, and muzzle geometry; `ProjectileDefinition` owns its collision, velocity, drag, terrain, and damage behavior. `World` owns instantiated entities and match state, including each tank's current resources and position, while `TerrainModel` owns the current Surface Heightmap. Physics therefore receives content, `World`, and `TerrainModel` explicitly instead of reading static gameplay constants or reconstructing instance values from defaults.
